Cybercrime Intelligence & Predictive Hotspot Engine for Response | SIH 2025

I. PRESENTATION LAYER (FRONTEND)
React/Vite Dashboard
The user interface for Law Enforcement Agencies (LEAs). Renders the map, filters, and alert lists. Handles user interactions (filtering, view actions) and state management.
‚Üí Logic Layer (API): Sends HTTP/JSON requests for fetching Alerts and Hotspots, and POST requests for triggering Complaint Processing.
‚Üí Real-Time Layer: Listens to the dedicated WebSocket channel for live updates.
React.js Vite Tailwind CSS React-Leaflet/Leaflet
Map Display
GIS-enabled component visualizing Reactive Alerts (markers) and Proactive Hotspots (risk heatmap circles).
SideBar Filters
Allows drill-down of displayed data by Priority Level and Time Range.
II. LOGIC LAYER (BACKEND / HYBRID AI ENGINE)
FastAPI Main API
Handles all routing, Pydantic validation, and dependency injection (DB Sessions).
‚Üí Presentation Layer: Responds with filtered JSON data (200 OK).
‚Üí Data Layer: Executes CRUD operations.
Python FastAPI SQLAlchemy Pydantic
Alert Aggregation Reactive Intelligence
Background task (/process-complaints) that processes new complaints, aggregates by city, calculates immediate risk/priority, and generates a simple trend-based AI Insight.
‚Üí Data Layer: Fetches raw complaints and saves derived alerts.
‚Üí Real-Time Layer: Triggers WebSocket broadcast for Critical Alerts.
Pandas (aggregation)
Predictive Engine Proactive Intelligence
Loads the trained ML model (text_risk_predictor.pkl) and runs inference on historical data (last 30-90 days) to predict future fraud likelihood/hotspots.
‚Üí Data Layer: Reads historical data for feature engineering and prediction.
Joblib Scikit-learn (ML) Pandas
III. REAL-TIME LAYER (WebSockets)
WebSocket Manager
Manages persistent connections (/ws/live) to LEA dashboards. Pushes new Critical Alert payloads asynchronously.
‚Üí Presentation Layer: Pushes real-time alert updates for immediate display.
‚Üí Logic Layer: Receives non-HTTP trigger from Alert Aggregation.
FastAPI WebSockets Python asyncio
IV. DATA & STORAGE LAYER
PostgreSQL Database
Stores normalized data: complaints (raw, translated text, financial details) and alerts (derived, aggregated risk summary).
‚Üí Logic Layer: Serves as the authoritative source for all data access (historical reads for ML, current reads for alerts, writes for new alerts).
PostgreSQL
Local Model Storage
Directory (/model) used to store the serialized, pre-trained ML model (text_risk_predictor.pkl) used by the Predictive Engine.
‚Üí Logic Layer: Loaded at application startup by the Predictive Engine.
Joblib Python
